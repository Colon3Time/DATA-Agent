KB อ่านตอนไหน — และ RAG ทำงานอย่างไร

  # orchestrator.py บรรทัด 218-237
  def load_relevant_kb(agent_name, task, top_n=6):   # ← อัปเกรด 2026-04-27: 4→6
      kb = load_kb(agent_name)       # อ่าน {agent}_methods.md + {agent}_decision_tree.md
      sections = re.split(r'\n(?=##)', kb)  # แบ่งตาม heading ##

      # TF-IDF cosine similarity
      docs = [task] + sections
      mat = TfidfVectorizer().fit_transform(docs)
      scores = cosine_similarity(mat[0:1], mat[1:])[0]
      top_idx = scores.argsort()[::-1][:top_n]  # เอา 6 sections ที่ตรงที่สุด
      return sections[top_idx]

ความหมาย: KB ไม่ถูกอ่านทั้งไฟล์ — ถูกอ่านเฉพาะ 6 sections ที่ตรงกับ task ที่สุด ผ่าน TF-IDF

  ดังนั้น heading ## ของแต่ละ section ใน KB มีผลต่อว่า section นั้นจะถูกดึงมาหรือเปล่า

  ---
  จุดที่ KB Upgrade มีผลจริง

  Task: "clean diabetes dataset, handle missing glucose values"
         ↓
  TF-IDF จับ keywords: diabetes, missing, glucose, values
         ↓
  ดึง sections ที่มี "MCAR/MAR/MNAR", "domain impossible zeros (Glucose=0)"
         ↓ (ก่อน upgrade — sections เหล่านี้ไม่มี)
  ดึงแค่ "Missing Values Method Selection" ทั่วไป

  สรุป: KB upgrade จะมีผลก็ต่อเมื่อ task keywords ตรงกับ heading/content ของ section ใหม่ใน KB — ถ้า task ไม่มี
  keyword ตรง TF-IDF ก็จะไม่ดึง section นั้นมาให้ agent
